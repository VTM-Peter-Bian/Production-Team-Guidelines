# VTM 版本號規範

**版本：** v0.1.0

---

請查看本次變更內容，依 SemVer 2.0 判斷應遞增的版本號。

## 任務

1. 檢視本次修改的範圍與影響
2. 建議新版本號（`MAJOR.MINOR.PATCH`）並說明理由
3. 若涉及發布，提醒用戶同步更新 `CHANGELOG.md`

## 遞增規則

| 段 | 場景 | 示例 |
|----|------|------|
| **MAJOR** | 不兼容變更、架構重構、公共 API 破壞性修改 | `1.0.0 → 2.0.0` |
| **MINOR** | 向後兼容的功能新增 | `1.2.3 → 1.3.0` |
| **PATCH** | Bug 修復、效能優化、小範圍調整 | `1.3.1 → 1.3.2` |

同一次發布含多種變更時，以最高優先級為準：**MAJOR > MINOR > PATCH**

## 客戶交付

- 首次正式交付客戶（APK、EXE 等）：固定 **1.0.0**
- 內部開發階段可從 `0.x.x` 起步

## 輸出格式

```
當前版本：（如已知）
建議版本：（新版本號）
遞增類型：MAJOR / MINOR / PATCH
理由：（簡要說明）

---
若確認發布，請同步更新 CHANGELOG.md。
```
