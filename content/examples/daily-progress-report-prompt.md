# VTM 每日專案進度匯報

**版本：** v0.1.0

---

請根據今日 GitHub Desktop 提交紀錄，生成可直接貼入 Planner 任務評論區的每日專案進度匯報。

## 任務

1. 檢視今日 GitHub Desktop 的 Commit 紀錄（Summary 與 Description）
2. 向用戶確認或接收以下資訊：
   - **序號**：本次匯報的一級序號（如 `5`）
   - **收件人**：匯報對象姓名，多人以逗號分隔（如 `Victor, Cindy`）
3. 依下方格式輸出完整匯報內容
4. 提醒用戶在 17:00 前貼入 Planner 對應任務的評論區

## 匯報格式

**一級標題（主標題）：**

```
{序號}) {收件人}, {yyyy-mm-dd} 每日專案進度匯報:
```

範例：`5) Victor, Cindy, 2026-07-08 每日專案進度匯報:`

**二級、三級條目：**

- 依今日 Commit 內容整理為分項，使用分級序號
- 二級：`{序號}.1)`、`{序號}.2)` …
- 三級：`{序號}.1.1)`、`{序號}.1.2)` …
- 每條對應一項具體工作成果，描述須清楚、可追溯

範例：

```
5) Victor, Cindy, 2026-07-08 每日專案進度匯報:
5.1) 完成 Module A01 場景載入問題修復…
5.2) 更新滅火器 3D 模型與貼圖資源…
5.3) 重構 Player Controller 腳本…
```

## 撰寫規則

- 日期使用當日 `yyyy-mm-dd`
- 內容須基於今日實際 Commit，不臆造未提交的工作
- 多個 Commit 可合併為相關分項，避免重複羅列
- 若 Commit 涉及程式與美術，分項說明須區分清楚
- 有支撐材料（APK、截圖、YouTube 影片等）時，在對應條目末尾標註引用，如 `attachments - [ 2. T&P ]`、`[ 3. Youtube ]`
- 避免模糊表述（如「更新代碼」「修復 bug」），須說明具體模組或內容

## 輸出格式

```
（輸出完整匯報內容，可直接貼入 Planner 評論區）

---
請確認內容無誤後，於 17:00 前貼入 Planner 對應任務的評論區。
```
