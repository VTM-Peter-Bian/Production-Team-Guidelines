字體大小設定

基準字級為 16px，行高 1.7。各層級字體大小如下：

序號 元素 字體大小 用途
1 正文 16px 頁面基準字級
2 欄目標題（h2） 2rem（32px） 各規範頁面主標題
3 區塊標題（h3） 1.125rem（18px） 內容小節標題
4 欄目描述 1rem（16px） 標題下方說明文字
5 表格內容 0.9rem（14.4px） spec-table 資料列
6 表格標題列 0.8rem（12.8px） spec-table 表頭
7 代碼範例 0.875rem（14px） 等寬字體程式碼區塊
8 輔助文字 0.9rem（14.4px） 注意事項、引用說明等
9 規範編號標籤 0.7rem（11.2px） 欄目分類標籤
10 側欄導航 0.9rem / 0.85rem 主導航 / 子導航連結

專案結構

文檔採用配置驅動 + 內容分離架構，避免所有內容堆疊在單一 index.html 中：

Production-Team-Guidelines/
├── index.html              # 頁面殼層（側欄 + 內容容器）
├── script.js               # 導航、內容載入、交叉引用、組織架構圖
├── styles.css              # 全域樣式
├── config/
│   └── nav.json            # 導航結構與欄目對應設定
├── content/
│   ├── bundle.js           # 打包檔（file:// 本地開啟必需，執行 build.py 生成）
│   ├── sections/           # 各欄目 HTML 片段（一欄一檔）
│   └── examples/           # 長篇範例（如 SDK CHANGELOG）
└── assets/                 # 圖片等靜態資源
    └── vendor/
        └── echarts.min.js  # 組織架構樹狀圖（進入對應欄目時懶加載）
build.py                    # 打包腳本

新增欄目步驟

序號 步驟 說明
1 建立內容檔 在 content/sections/ 新增 {id}.html，包含完整 <article> 結構
2 註冊欄目 在 config/nav.json 的 nav 與 sections 中新增對應項目
3 長篇範例 過長的參考示例放入 content/examples/，以 data-example 按需載入
4 交叉引用 欄目間連結使用 section-ref 類別與 data-section 屬性
5 重新打包 修改內容後執行 python3 build.py 更新 content/bundle.js，確保本地直接開啟可用

欄目 HTML 結構

每個欄目檔案須包含完整 <article>，並設定 id 與 data-section（值與欄目 ID 一致）：

序號 區塊 類別 / 元素 用途
1 欄目根節點 article.section 含 id、data-section；特殊樣式可加 section-delivery、section-reds 等
2 標題區 header.section-header 含 section-label、h2、section-desc
3 內容小節 div.content-block 每個 h3 小節包在獨立 content-block 中
4 子項錨點 div.habit-block / div.topic-block 欄目內可跳轉的子區塊，設 id（如 habit-01、planner-task-fields）供交叉引用；標題用 habit-title / topic-title 搭配編號徽章
5 注意事項 aside.note-box 置於欄目末尾，含 h4 與說明段落

圖片與截圖規範

操作示意、介面截圖等靜態圖片統一存放於 assets/，並使用標準的截圖展示結構：

序號 規範 說明
1 存放路徑 圖片放於 assets/，引用路徑為 assets/檔名.png（相對於專案根目錄）
2 檔名命名 小寫英文與連字號，語意清楚，如 outlook-reaction.png、teams-reaction.png
3 檔案格式 優先使用 PNG；需透明背景時用 PNG，照片類可用 JPG（品質 ≥ 80%）
4 尺寸與壓縮 寬度建議 ≤ 1200px；上傳前壓縮，單檔盡量 < 500 KB，避免拖慢載入
5 無障礙文字 每張圖須設 alt，簡述畫面內容與用途，勿留空
6 延遲載入 加上 loading="lazy"，減少首屏負擔
7 截圖結構 使用 figure.guide-screenshot 包裹，內含 guide-screenshot-media 與 figcaption 說明
8 多圖排列 並排展示時外層用 div.guide-screenshots（響應式網格，窄螢幕自動單欄）

標準 HTML 範例（參見 工作習慣 01）：

<div class="guide-screenshots">
  <figure class="guide-screenshot">
    <div class="guide-screenshot-media">
      <img src="assets/example.png"
           alt="操作步驟示意說明"
           loading="lazy">
    </div>
    <figcaption>圖片下方簡短說明</figcaption>
  </figure>
</div>

組織架構樹狀圖

層級型組織架構圖使用 Apache ECharts 的 tree 系列繪製，避免手寫 CSS 連線或流程圖工具自動排版造成的錯位。參見 組織架構。

序號 項目 說明
1 依賴檔案 assets/vendor/echarts.min.js，僅在含圖表的欄目載入時注入，不影響其他頁面
2 HTML 掛載點 欄目內放置 <div id="org-chart-mount" class="org-chart-mount">，並設 role="img" 與 aria-label
3 樹狀資料 在 script.js 的 ORG_CHART_DATA 維護父子層級；切換欄目時自動 dispose 圖表實例
4 版面與連線 自上而下（orient: "TB"）、折線連接（edgeShape: "polyline"），SVG 渲染以利縮放清晰
5 節點樣式 公司根節點黑底白字；重點部門白底黑框；一般技術單元灰底；CEO 分管部門使用 ORG_CHART_MUTED（虛線、淡色文字）
6 換行 節點標籤以 \n 換行；透過 symbolSize: [寬, 高] 控制方框尺寸
7 容器樣式 外層使用 .org-chart-mount（定義於 styles.css），最小高度確保樹狀圖完整顯示
8 頁面說明 欄目正文只描述架構含義，勿標注繪圖工具或第三方庫名稱

欄目 HTML 掛載點範例：

<div id="org-chart-mount" class="org-chart-mount"
     role="img" aria-label="VTM 公司組織架構樹狀圖"></div>

更新組織架構時，修改 script.js 中 ORG_CHART_DATA 的 children 層級即可；無需改動欄目 HTML（除非調整說明表格或文案）。

長篇範例載入

過長內容不直接內嵌於欄目 HTML，改放 content/examples/ 並按需載入：

序號 屬性 適用場景 示例欄目
1 data-example 純文字 / Markdown 範例，載入後填入元素內文 CHANGELOG 規範
2 data-example-html HTML 片段範例，載入後替換容器 innerHTML 職安培訓交付標準

內容撰寫規範

序號 規範 說明
1 表格須含序號欄 所有表格第一欄為序號（參見 工作習慣 02）
2 交叉引用 相關規範之間使用 section-ref 連結，點擊可跳轉至對應欄目
3 欄目分類 section-label 須與側欄導航分組名稱一致；頂層獨立欄目使用其導航標籤；頁腳欄目使用「附錄」
4 注意事項區塊 重要提醒使用 note-box 呈現，置於欄目末尾
5 持續更新 新增欄目時同步更新 config/nav.json 與交叉引用
6 通用優先 規範正文寫通用原則；特定項目（如 SDK）的示例獨立為「參考示例」區塊
7 圖片與截圖 操作示意須附截圖，遵循上方「圖片與截圖規範」的結構與命名要求
8 組織架構圖 層級架構使用 ECharts 樹狀圖，遵循上方「組織架構樹狀圖」規範；勿以純 CSS 手繪或流程圖語法替代
9 版本與日期 重大更新時同步修改 config/nav.json 的 site.version 與 site.updatedAt

注意事項

修改樣式時請同步更新本文檔中的字體大小設定表，確保規範與實際呈現一致。發布新版本時，請更新側欄版本號、更新日期，以及本文「基本資訊」表中的對應欄位。
