SharePoint 項目管理

SharePoint Documents 目錄規範

各項目於 SharePoint Documents 下依階段編號建立資料夾，統一存放項目文件與交付物。上傳資源前須確認目標資料夾，避免混放或覆蓋他人檔案。

目錄結構總覽

每個項目在 SharePoint Documents 根目錄下，依項目生命週期劃分為編號資料夾（0–7）。開發階段（4）再依職能細分為 Graphics、Material、Programming 三個子資料夾。

序號 資料夾 用途 主要存放內容
0 Project Management 項目管理 T&P（Task & Progress）、會議紀錄、項目計劃與管理類文件
1 Requirement 需求階段 需求文件、客戶確認材料、需求變更記錄
2 Design 設計階段 設計稿、流程圖、原型與設計評審材料
3 Planning 規劃階段 排期、資源規劃、里程碑計劃
4 Development - Graphics 美術開發 Graphic Team 產出的 APK、EXE、進度截圖 / 圖片、美術資源包等
4 Development - Material 生產材料 項目生產所用到的原材料與參考素材（模型源檔、貼圖源檔、音效、參考圖等）
4 Development - Programming 程式開發 Program Team 產出的 APK、EXE、進度截圖 / 圖片、測試包等
5 Testing 測試階段 測試報告、Bug List、測試用安裝包與驗證材料
6 Deployment 部署交付 正式交付客戶的安裝包、部署說明、交付清單（首次交付版本號為 v1.0.0，見 版本號規範）
7 Operation & Maintenance 運維階段 上線後維護記錄、客戶反饋處理、迭代更新包

開發階段（4）分工對照

開發過程中產出的安裝包與進度材料，須依負責團隊上傳至對應子資料夾，不可混放：

序號 團隊 目標資料夾 典型檔案
1 Graphic Team 4 Development - Graphics APK、EXE、場景 / UI 進度截圖、渲染預覽圖、美術資源匯出包
2 Program Team 4 Development - Programming APK、EXE、功能演示截圖、除錯用測試包、整合進度材料
3 全員（生產素材） 4 Development - Material 項目製作所需的原始素材與參考資料，非每日進度匯報包

影片類檔案因 SharePoint 儲存空間限制，不上傳至此目錄，改依 YouTube 上傳和分享規範 處理，並於 Planner 附上連結。

日常操作要點

序號 事項 說明
1 更新 T&P 每日到崗與下班前更新 T&P，檔案存放於 0 Project Management（詳見 每日工作流程）
2 進度材料歸檔 APK、EXE、截圖等進度支撐材料上傳至對應的 4 Development - Graphics 或 4 Development - Programming，並於 Planner 任務中附上 SharePoint 連結
3 檔名與版本 安裝包檔名須能辨識項目與版本號；首次交付客戶為 v1.0.0
4 避免覆蓋 同一檔案有新版本時，以版本號或日期區分檔名，保留歷史記錄，勿直接覆蓋舊檔

注意事項

SharePoint 為項目文件的權威存放位置；源代碼與美術工程檔仍須依 GitHub Desktop 提交規範 提交至 GitHub。兩者分工：SharePoint 存放可分享的交付物與進度材料，GitHub 存放可版本控制的源代碼與工程資源。
