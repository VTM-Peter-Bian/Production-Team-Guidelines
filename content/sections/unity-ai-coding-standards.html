Unity C# AI 編程規範

供 Cursor / AI 輔助編程使用的 VTM Unity C# 守則。可一鍵複製完整 Prompt 至專案 Rules 或對話開頭。

核心規則摘要

序號 類別 規則
1 命名 新腳本以 VTM 開頭 + PascalCase；類名與檔名一致
2 目錄 置於 Assets/VTM/Scripts/{Core|Functions|UI|Data|Editor}/
3 命名空間 VTM.{模組名稱}，與目錄結構一致
4 序列化 使用 [SerializeField] private，避免 public 欄位
5 註釋 公開方法須有 XML 註釋（///）
6 性能 Awake 快取引用；禁止 Update 內 GetComponent / Find；避免 GC 分配
7 日誌 前綴含類名；正式版以條件編譯禁用 Debug
8 數據 配置抽離至 ScriptableObject / JSON，禁止魔術數字與硬編碼字串

目錄結構

Assets/
└── VTM/
    └── Scripts/
        ├── Core/       # 核心系統腳本
        ├── Functions/  # 遊戲功能腳本
        ├── UI/         # UI 相關腳本
        ├── Data/       # 資料結構和 ScriptableObject
        └── Editor/     # 編輯器擴展腳本

命名後綴對照

序號 後綴 用途 範例

4 Panel / Button UI 元件 VTMSettingsPanel.cs

使用方式

序號 方式 說明
1 專案 Rules 複製 Prompt 至專案 .cursor/rules，作為持久約束
2 任務開頭引用 在 Composer / Chat 開頭貼上 Prompt，並以 @Folders Assets/VTM 提供上下文
3 Code Review 對照本規範與 程式架構評分等級 審查 AI 產出

注意事項

本規範專注於 VTM 專案腳本的命名、結構與 AI 協作品質。架構評審請對照 REDS 術語；Cursor 操作能力評核請對照 Cursor AI 評核標準。版本變更請同步更新 content/examples/unity-ai-coding-prompt.md 並執行 python3 build.py。
