規範 01

每日工作流程

開發組成員每日到崗、離崗的標準工作節奏與匯報要求。

10:00

到達公司，開始工作

到崗後依序完成以下事項：

  • 在公司門口打卡機刷臉打卡
  • 查看 Outlook 郵件與行事曆，並按 👍 標記已讀（詳見 工作習慣 01）
  • 瀏覽 SharePoint 最新資訊
  • 查看 Teams 訊息，並按 👍 標記已讀（詳見 工作習慣 01）
  • 更新 T&P（Task & Progress，用於管理同事的工作進度）
  • 回覆相關訊息與待辦內容（須標記序號，詳見 工作習慣 03）
13:00 – 14:00

午飯時間

午休時段，暫停工作事項。

17:00 前

每日進度匯報

下班前須完成以下匯報工作：

  • 更新 T&P（Task & Progress）
  • 在 Planner 對應的 Task 上進行每日工作進度匯報
  • 附上支撐材料，例如：
    • APK 安裝包
    • 截圖 / 圖片
    • 錄影 / 影片（詳見 YouTube 上傳和分享規範）
18:00 前

工作成果提交

使用 GitHub Desktop 將當日工作成果提交至 GitHub，詳見 GitHub Desktop 提交規範：

  • Program Team：提交當日代碼更變
  • Graphic Team：提交當日製作的美術資源（模型、貼圖、UI 素材等）
18:00

下班

完成當日工作後，依序完成以下事項再離開公司：

序號 事項 說明
1 關閉設備 關閉個人電腦及顯示屏
2 刷臉打卡 離開前在公司門口打卡機刷臉打卡
3 關閉 Logo 燈 18:00 下班後第一位離開公司的人，請關掉公司 Logo 燈
4 檢查座位區域 如附近座位同事已離開，請確保所屬座位附近的燈光及冷氣已全部關閉
5 最後離開檢查 如為全公司最後一位下班，請再次檢查所有燈光及冷氣已關閉，並鎖上大門

注意事項

進度匯報須有明確的支撐材料，確保工作成果可追溯、可驗證。工作成果提交（含 Program Team 代碼與 Graphic Team 美術資源）請在 18:00 前完成，避免累積未推送的變更。18:00 準時下班，離開前請完成刷臉打卡，並關閉個人電腦及顯示屏。
