規範 04 · 源代碼與版本控制

GitHub Desktop 提交規範

使用 GitHub Desktop 進行日常代碼與美術資源提交、推送的操作標準。

帳戶註冊

  • 使用公司提供的個人郵箱進行註冊（帳戶資料設定詳見 GitHub 帳戶規範）

適用對象與提交內容

序號 團隊 提交內容 示例
1 Program Team 當日代碼更變 腳本、場景、Prefab、設定檔等
2 Graphic Team 當日美術資源 3D 模型、貼圖、材質、UI 素材、動畫等

兩個團隊均須在每日 18:00 前完成提交並推送，避免工作成果僅留存在本地。

Commit Summary 格式

格式：

yyyy-mm-dd Content Summary...

範例：

2026-07-07 Update VTM Unity SDK

2026-07-08 Update fire extinguisher 3D models and textures

Commit Description

以 list 方式列出本次提交的具體內容。Program Team 範例：

- Update VTM Unity SDK to v2.1.0
- Fix scene loading issue in Module A01
- Refactor player controller script

Graphic Team 範例：

- Add fire extinguisher 3D model (FBX)
- Update wall texture maps for Module A01
- Export UI button assets for training menu

注意事項

請在每日 18:00 前完成當日工作成果提交（Program Team 代碼、Graphic Team 美術資源，參閱 每日工作流程）。Summary 須包含日期與簡要說明，Description 須清楚列出各項變更，便於追溯與審閱。SDK 版本發布時須同步遵循 版本號規範 與 CHANGELOG 規範。
