源代碼與版本控制

CHANGELOG 規範

CHANGELOG.md 記錄各交付物版本的重大變更，按版本倒序排列，並以「分類 + 路徑」標記變更位置。

適用範圍

凡採用 版本號規範 管理的交付物，均須維護對應的 CHANGELOG.md，包括但不限於：

序號 交付物 CHANGELOG 位置
1 VTM Unity SDK SDK 根目錄 CHANGELOG.md
2 培訓項目 Repository 項目根目錄 CHANGELOG.md 或 Documents/Change_Log.md
3 內部工具 工具 Repository 根目錄 CHANGELOG.md

撰寫原則

序號 原則 說明
1 版本倒序 最新版本置頂，依時間由新到舊排列
2 分類標記 每條變更以 【分類名】 標記（如模組名、功能領域、子系統）
3 路徑標記 變更涉及的目錄或檔案路徑使用 反引號 標記
4 變更類型 使用 Added、Changed、Fixed 三類分組
5 參考標準 基於 語義化版本控制 與 約定式提交 規範

條目格式

每條變更條目格式如下：

- 【分類名】變更描述（含路徑，使用 `反引號` 標記）

通用範例：

- 【核心系統】重構 `Services/Auth` 登入流程，支援多因素驗證
- 【培訓場景】新增 `Scenes/Level03` 工廠火災演練關卡
- 【文件】更新 `Documents/User_Manual` 操作說明
- 【打包工具】修復 `Editor/BuildPipeline` 在 macOS 下的路徑識別問題

文件結構範例

以下為通用的 CHANGELOG.md 結構模板，適用於各類交付物：

# Changelog

本文件記錄 [交付物名稱] 的重大變更。
基於 [語義化版本控制](https://semver.org/spec/v2.0.0.html) 與 [約定式提交](https://www.conventionalcommits.org/) 規範。

## [1.2.0] - 2026-03-15

### Added
- 【功能模組】新增 …

### Changed
- 【配置調整】調整 …

### Fixed
- 【Bug 修復】修復 …

注意事項

每次發布新版本時，須在 CHANGELOG.md 頂部新增對應版本區塊，並依 版本號規範 決定版本號遞增類型。變更描述須具體到分類與路徑，避免僅寫「修復 bug」等模糊表述。
