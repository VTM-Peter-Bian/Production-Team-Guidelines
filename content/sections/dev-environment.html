入門與日常

開發組環境

開發組以 Unity 6 為核心進行 VR 項目製作；周邊涵蓋雲端資料、源代碼管理、目標平台、第三方整合、AI 協作工具與內部軟體工廠。

環境總覽圖

核心分支一覽

開發組環境以 Unity 6 為核心，僅有六個一級核心分支直接連接 Unity；其餘組件均歸屬於對應分支。

5 AI AI 協作開發工具與 API；含子分支 5.1 Tools、5.2 API
6 3rd Party 第三方 Plugin、SDK、Hardware；含子分支 6.1 Plugin、6.2 SDK、6.3 Hardware

1. Data Management（資料管理）

透過 Azure 雲端服務管理項目資料、資料庫與檔案儲存；Unity 客戶端透過 API 讀寫遠端資料。

序號 組件 類型 狀態 說明
1 Data Management 核心分支 使用中 資料管理總類；匯入 Unity 的雲端資料服務
1.1 Azure Static Web 組成部分 使用中 透過 API 建立可視化資料管理介面，支援 CRUD 操作
1.2 Azure Database 組成部分 使用中 Azure Database Flexible Server 遠端 MySQL；開發／測試／正式環境分別對應 VTM-FRP、VTM-CPP
1.3 AWS Web Services 棄用 棄用 舊版 Web 管理介面，已由 Azure Static Web 取代
1.4 AWS Database 棄用 棄用 舊版遠端 MySQL，已由 Azure Database 取代
1.5 Azure Blob Storage 組成部分 使用中 彈性、安全且具成本效益的雲端儲存（vtm_storage）
1.6 AWS S3 Storage 棄用 棄用 舊版雲端儲存，已由 Azure Blob Storage 取代

2. Source Code & Version Control（源代碼與版本控制）

程式碼與美術資源於項目進行中統一存放於 GitHub，日常提交使用 GitHub Desktop。項目完成後 3 個月，PO 會將 Repository 從 GitHub 移除，並由 Back Up 流程歸檔至公司 VTM-NAS。詳見 GitHub Desktop 提交規範。

序號 組件 類型 狀態 說明
2 Source Code & Version Control 核心分支 使用中 源代碼與版本控制總類；匯入 Unity 的代碼託管服務
2.1 GitHub Desktop 組成部分 使用中 圖形化版本控制客戶端，用於日常 commit 與 push
2.2 GitHub 組成部分 使用中 代碼託管、版本控制、團隊協作、Issue 追蹤與權限管理（項目生命週期內）
2.3 Back Up 核心分支 使用中 項目完成後的源代碼歸檔分支；PO 於完成後 3 個月 將項目從 GitHub 移除，並備份至 VTM-NAS
2.4 VTM-NAS 組成部分 使用中 公司網路儲存（NAS），存放已從 GitHub 下架的項目完整備份
2.5 Smart SVN 棄用 棄用 舊版圖形化 SVN 客戶端
2.6 SVN 棄用 棄用 舊版版本控制系統，已由 GitHub 取代

3. Platforms（目標平台）

Unity 構建產出部署至 Android、Windows 與 iOS 平台；VR 一體機透過 OpenXR 標準接入。PC 端 Application 可整合 3rd Party 提供的 Plugin 與 SDK（如 OpenCV、Omni One SDK）。

序號 組件 類型 說明
3 Platforms 核心分支 目標平台總類；Unity 構建產出的部署方向
3.1 Android 核心分支 獨立 APK 與 VR 一體機構建目標
3.2 XR Headset 組成部分 Android VR 一體機頭顯適配（Meta Quest、Pico、HTC）
3.3 Meta Quest 組成部分 Quest 系列頭顯
3.4 Pico 組成部分 Pico 系列頭顯
3.5 HTC 組成部分 HTC Vive 系列頭顯
3.6 API Standard 組成部分 Android 端 XR API 抽象層
3.7 OpenXR 組成部分 跨平台 XR 標準；Meta Quest／Pico／HTC、API Standard 與 PCVR 均接入
3.8 Windows 核心分支 桌面應用與 PCVR 構建目標
3.9 Application 組成部分 一般 Windows 應用程式；整合 3rd Party Plugin／SDK（如 OpenCV、Omni One SDK）
3.10 PCVR 組成部分 PC 端 VR 體驗
3.11 iOS 核心分支 iOS 平台構建目標

4. VTM Software Factory（內部軟體工廠）

開發組自研的共用資產工廠，劃分為 Library、Standard、Template、One-Off 四個子分支。Library 依 TSA／REDS（Standard）建構；CTP／PMTP／XRA 與 One-Off 均以虛線關聯至 Library 與 Standard。跨分支以虛線連至彙總節點。

序號 組件 類型 說明
4 VTM Software Factory 核心分支 內部軟體工廠總類
4.1 Library 核心分支 共用程式庫子分支；依 Standard（TSA／REDS）建構
4.1.1 Design Library 組成部分 設計相關共用資源與工具
4.1.2 Function Library 組成部分 通用功能模組
4.1.3 UI Library 組成部分 共用 UI 元件與樣式
4.2 Standard 核心分支 開發標準與架構原則子分支
4.2.1 TSA 組成部分 Task · Step · Action 流程拆解標準
4.2.2 REDS 組成部分 Reusable · Expandable · Data-Driven · Standardization 架構指標
4.3 Template 核心分支 項目模板子分支；虛線關聯至 Library 與 Standard
4.3.1 CTP 組成部分 項目模板；虛線關聯至 Library 與 Standard
4.3.2 PMTP 組成部分 項目模板；虛線關聯至 Library 與 Standard
4.3.3 XRA 組成部分 項目模板；虛線關聯至 Library 與 Standard
4.4 One-Off 核心分支 一次性／專案特製資產子分支；虛線關聯至 Library 與 Standard

5. AI（人工智慧協作）

開發組使用 AI 工具輔助程式撰寫、研究與文檔整理；分為Tools（桌面／網頁工具）與 API（模型服務接口）兩類。Cursor 使用規範詳見 Cursor AI 評核標準；Prompt 彙總見 AI Prompt 一覽。

序號 組件 類型 狀態 說明
5 AI 核心分支 使用中 AI 協作開發總類；輔助 Unity 項目開發與日常文檔工作
5.1 Tools 核心分支 使用中 AI 協作工具子分支
5.1.1 Cursor 組成部分 使用中 IDE 內嵌 AI 編程助手，用於 C#／Unity 日常開發
5.1.2 Genspark 組成部分 使用中 AI 搜尋與研究輔助工具
5.1.3 MiniMax 組成部分 使用中 AI 對話與內容生成工具
5.2 API 核心分支 使用中 AI 模型 API 子分支
5.2.1 DeepSeek 組成部分 使用中 DeepSeek 大語言模型 API
5.2.2 Doubao 組成部分 使用中 豆包（字節跳動）大語言模型 API

6. 3rd Party（第三方整合）

外部廠商提供的 Plugin、SDK 與 Hardware，以 Unity Package 或原生庫形式整合至項目。硬體與 SDK 通常成對出現（如 Omni One／Omni One SDK、PortalCam／LCC Unity）；Webcam 連接 OpenCV Plugin；SDK／Plugin 再跨分支連接 3.9 Application。

序號 組件 類型 狀態 說明
6 3rd Party 核心分支 使用中 第三方整合總類；Plugin、SDK、Hardware 匯入 Unity
6.1 Plugin 核心分支 使用中 第三方 Unity Plugin 子分支
6.1.1 OpenCV 組成部分 使用中 電腦視覺 Plugin；連接 Webcam 硬體，並跨分支連接 PC Application
6.2 SDK 核心分支 使用中 第三方廠商 SDK 子分支
6.2.1 Omni One SDK 組成部分 使用中 Omni One 廠商 SDK；與 Omni One 硬體配對，連接 PC Application 開發 PCVR 內容
6.2.2 LCC Unity 組成部分 使用中 PortalCam 配套 Unity SDK；與 PortalCam 硬體配對，連接 PC Application
6.2.3 Votanic 組成部分 使用中 第三方 Unity SDK；連接 Windows Application
6.3 Hardware 核心分支 使用中 第三方硬體子分支
6.3.1 Omni One 組成部分 使用中 第三方 PCVR 硬體；與 Omni One SDK 配對
6.3.2 Webcam 組成部分 使用中 PC 外接攝像頭；連接 OpenCV Plugin，用於 PC Application
6.3.3 PortalCam 組成部分 使用中 第三方攝像／採集硬體；與 LCC Unity SDK 配對
