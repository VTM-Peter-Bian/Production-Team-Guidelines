入門與日常

開發組環境

開發組以 Unity 6 為核心進行項目製作；周邊涵蓋雲端資料、源代碼管理、目標平台、第三方整合、AI 協作工具、內部軟體工廠、開發工具與開發硬體。

核心分支一覽

開發組環境以 Unity 6 為核心，僅有九個一級核心分支直接連接 Unity；序號以圖中位置為準，自 Software Factory 起順時針編為 1 至 9。

序號 核心分支 說明
1 VTM Software Factory 開發組自研軟體工廠；含子分支 Library、Standard、Template、One-Off
2 Hardware 開發組工作用硬體；Mac Mini、Mac Server、Workstation、iPad 分別虛線關聯 VTM-NAS、SVN、Windows、app
3 OpenXR 跨平台 XR 標準；下接廠商 SDK，再至 VR 一體機硬體（虛線關聯 apk）
4 OS Unity 構建產出的目標作業系統（Android、Windows、iOS）
5 3rd Party 第三方 Plugin、SDK；硬體緊接於對應 Plugin／SDK 之後（如 OpenCV → Webcam、Omni One SDK → Omni One）
6 Dev Tools 美術與程式開發工具；含子分支 Graphic（2D／3D）、Design（VO）、Program（Code 虛線關聯 Cursor）
7 AI AI 協作開發工具與 API；含子分支 7.1 Tools、7.2 API
8 Source & Version Control GitHub 託管與日常提交（含程式與美術資源）；含子分支 8.3 Back Up（完成後 3 個月歸檔至 VTM-NAS）
9 Data Management 雲端資料、資料庫與檔案儲存管理

1. VTM Software Factory（內部軟體工廠）

開發組自研的共用資產工廠，劃分為 Library、Standard、Template、One-Off 四個子分支。Library 下分 Internal（內部）與 Production（生產）；生產分支含 Design Library、Function Library、Graphic Library。Library 依 TSA／REDS（Standard）建構；CTP／PMTP／XRA 與 One-Off 均以虛線關聯至 Library 與 Standard。

序號 組件 類型 說明
1 VTM Software Factory 核心分支 內部軟體工廠總類
1.1 Library 核心分支 共用資產庫子分支；分 Internal（內部）與 Production（生產）
1.1.1 Internal 核心分支 內部用共用資產庫
1.1.2 Production 核心分支 生產用共用資產庫
1.1.2.1 Design Library 組成部分 設計相關共用資源與工具
1.1.2.2 Function Library 組成部分 通用功能模組
1.1.2.3 Graphic Library 組成部分 美術／圖形相關共用資源
1.2 Standard 核心分支 開發標準與架構原則子分支
1.2.1 TSA 組成部分 Task · Step · Action 流程拆解標準
1.2.2 REDS 組成部分 Reusable · Expandable · Data-Driven · Standardization 架構指標
1.3 Template 核心分支 項目模板子分支；虛線關聯至 Library 與 Standard
1.3.1 CTP 組成部分 項目模板；虛線關聯至 Library 與 Standard
1.3.2 PMTP 組成部分 項目模板；虛線關聯至 Library 與 Standard
1.3.3 XRA 組成部分 項目模板；虛線關聯至 Library 與 Standard
1.4 One-Off 核心分支 一次性／專案特製資產子分支；虛線關聯至 Library 與 Standard

2. Hardware（開發硬體）

開發組日常工作用電腦、伺服器與測試裝置；Mac Mini 虛線關聯 VTM-NAS（歸檔存取），Mac Server 虛線關聯 SVN（舊版版本庫），Workstation 虛線關聯 Windows 平台構建，iPad 虛線關聯 app（iOS 構建產物）。

序號 組件 類型 說明
2 Hardware 核心分支 開發硬體總類
2.1 Mac Mini 組成部分 開發用 Mac Mini；虛線關聯 VTM-NAS
2.2 Mac Server 組成部分 開發用 Mac 伺服器；虛線關聯 SVN
2.3 Workstation 組成部分 Windows 開發工作站；虛線關聯 Windows 平台
2.4 iPad 組成部分 iOS 測試與展示用平板；虛線關聯 app

3. OpenXR（跨平台 XR 標準）

與 OS 同為 Unity 6 的一級核心分支。下接廠商 SDK（Meta XR SDK、Pico SDK、VIVE OpenXR），再至對應 VR 一體機硬體系列；硬體以虛線關聯 4.1.1 apk（Android 構建產物）。

序號 組件 類型 說明
3 OpenXR 核心分支 跨平台 XR 標準；直接連接 Unity 6
3.1 Meta XR SDK 組成部分 Meta Quest 廠商 SDK；下接 Meta Quest Series
3.1.1 Meta Quest Series 組成部分 Quest 系列頭顯；虛線關聯 apk
3.2 Pico SDK 組成部分 Pico 廠商 SDK；下接 Pico Series
3.2.1 Pico Series 組成部分 Pico 系列頭顯；虛線關聯 apk
3.3 VIVE OpenXR 組成部分 HTC 提供的 OpenXR 廠商擴展 SDK（非 OpenXR 標準本體）；下接 HTC VIVE Series
3.3.1 HTC VIVE Series 組成部分 HTC Vive 系列頭顯；虛線關聯 apk

4. OS（目標作業系統）

Unity 構建產出部署至 Android、Windows 與 iOS；Android 下接構建產物 apk，iOS 下接 app。PCVR 分為兩類：Native PCVR（PC 端 exe 直連頭顯）與 PC + Standalone VR（透過 Steam Link、Meta Horizon Link 等串流至一體機）。PC 端 Application 可整合 3rd Party 提供的 Plugin 與 SDK。

序號 組件 類型 說明
4 OS 核心分支 目標作業系統總類；Unity 構建產出的部署方向
4.1 Android 核心分支 Android 平台構建目標
4.1.1 apk 組成部分 Android 構建產物（獨立 APK）；XR 一體機硬體以虛線關聯
4.2 Windows 核心分支 桌面應用與 PCVR 構建目標
4.2.1 Application 組成部分 一般 Windows 應用程式；整合 3rd Party Plugin／SDK（如 OpenCV、Omni One SDK）
4.2.2 PCVR 核心分支 PC 端 VR 體驗子分支
4.2.2.1 Native PCVR 組成部分 PC 端 exe 運行，頭顯與控制器直連 PC（如 VIVE 有線／接收器）；虛線關聯 OpenXR、HTC VIVE Series
4.2.2.2 PC + Standalone VR 組成部分 PC 端 exe 渲染，透過 Steam Link、Meta Horizon Link 等串流至一體機顯示並回傳追蹤／輸入；虛線關聯 Meta Quest Series、Pico Series
4.3 iOS 核心分支 iOS 平台構建目標
4.3.1 app 組成部分 iOS 構建產物（.app / App Store 發布包）；iPad 以虛線關聯

5. 3rd Party（第三方整合）

外部廠商提供的 Plugin 與 SDK，以 Unity Package 或原生庫形式整合至項目。對應硬體緊接於各 Plugin／SDK 之後（如 OpenCV → Webcam、Omni One SDK → Omni One、LCC Unity SDK → PortalCam、Votanic SDK → CAVE）；SDK／Plugin 再跨分支連接 4.2.1 Application。

序號 組件 類型 狀態 說明
5 3rd Party 核心分支 使用中 第三方整合總類；Plugin、SDK 匯入 Unity，硬體緊接於對應 Plugin／SDK
5.1 Plugin 核心分支 使用中 第三方 Unity Plugin 子分支
5.1.1 OpenCV 組成部分 使用中 電腦視覺 Plugin；下接 Webcam 硬體，並跨分支連接 PC Application
5.1.2 Webcam 組成部分 使用中 PC 外接攝像頭；緊接 OpenCV Plugin，用於 PC Application
5.2 SDK 核心分支 使用中 第三方廠商 SDK 子分支
5.2.1 Omni One SDK 組成部分 使用中 Omni One 廠商 SDK；下接 Omni One 硬體，連接 PC Application 開發 PCVR 內容
5.2.2 LCC Unity SDK 組成部分 使用中 PortalCam 配套 Unity SDK；下接 PortalCam 硬體，連接 PC Application
5.2.3 Votanic SDK 組成部分 使用中 第三方 Unity SDK；下接 CAVE 硬體，連接 Windows Application
5.2.4 Omni One 組成部分 使用中 第三方 PCVR 硬體；緊接 Omni One SDK
5.2.5 PortalCam 組成部分 使用中 第三方攝像／採集硬體；緊接 LCC Unity SDK
5.2.6 CAVE 組成部分 使用中 沉浸式投影硬體；緊接 Votanic SDK

6. Dev Tools

開發組日常美術與程式製作所使用的桌面工具；Graphic 分支涵蓋 2D／3D 設計軟體，Design 下含 VO（虛線關聯 7.1.4 MiniMax），Program 下含 Code，以虛線關聯 7.1.1 Cursor、7.1.2 Claude Code（AI Tools）；Genspark 亦以虛線關聯 Graphic。

序號 組件 類型 說明
6 Dev Tools 核心分支 開發工具總類
6.1 Graphic 核心分支 美術設計工具子分支
6.1.1 2D 核心分支 平面設計工具
6.1.1.1 Affinity 組成部分 2D 設計與排版
6.1.1.2 Figma 組成部分 UI／UX 協作設計
6.1.2 3D 核心分支 三維建模與貼圖工具
6.1.2.1 Blender 組成部分 3D 建模與動畫
6.1.2.2 Substance Painter 組成部分 3D 材質與貼圖繪製
6.2 Design 核心分支 設計輔助工具子分支
6.2.1 VO 組成部分 語音／視覺設計輔助；虛線關聯 MiniMax（AI Tools）
6.3 Program 核心分支 程式開發工具子分支
6.3.1 Code 組成部分 程式碼編輯；虛線關聯 Cursor、Claude Code（AI Tools）

7. AI（人工智慧協作）

開發組使用 AI 工具輔助程式撰寫、研究與文檔整理；分為Tools（桌面／網頁工具）與 API（模型服務接口）兩類。Cursor 使用規範詳見 Cursor AI 評核標準；Prompt 彙總見 AI Prompt 一覽。

序號 組件 類型 狀態 說明
7 AI 核心分支 使用中 AI 協作開發總類；輔助 Unity 項目開發與日常文檔工作
7.1 Tools 核心分支 使用中 AI 協作工具子分支
7.1.1 Cursor 組成部分 使用中 IDE 內嵌 AI 編程助手，用於 C#／Unity 日常開發
7.1.2 Claude Code 組成部分 使用中 終端 AI 編程助手；虛線關聯 Code、Graphic（Dev Tools）
7.1.3 Genspark 組成部分 使用中 AI 搜尋與研究輔助工具
7.1.4 MiniMax 組成部分 使用中 AI 對話與內容生成工具
7.2 API 核心分支 使用中 AI 模型 API 子分支
7.2.1 DeepSeek 組成部分 使用中 DeepSeek 大語言模型 API
7.2.2 Doubao 組成部分 使用中 豆包（字節跳動）大語言模型 API

8. Source & Version Control（源與版本控制）

程式碼與美術資源於項目進行中統一存放於 GitHub，日常提交使用 GitHub Desktop。項目完成後 3 個月，PO 會將 Repository 從 GitHub 移除，並由 Back Up 流程歸檔至公司 VTM-NAS。詳見 GitHub Desktop 提交規範。

序號 組件 類型 狀態 說明
8 Source & Version Control 核心分支 使用中 源與版本控制總類；程式與美術資源託管服務
8.1 GitHub Desktop 組成部分 使用中 圖形化版本控制客戶端，用於日常 commit 與 push
8.2 GitHub 組成部分 使用中 代碼託管、版本控制、團隊協作、Issue 追蹤與權限管理（項目生命週期內）
8.3 Back Up 核心分支 使用中 項目完成後的源代碼歸檔分支；PO 於完成後 4 個月 將項目從 GitHub 移除，並備份至 VTM-NAS
8.4 VTM-NAS 組成部分 使用中 公司網路儲存（NAS），存放已從 GitHub 下架的項目完整備份
8.5 Smart SVN 棄用 棄用 舊版圖形化 SVN 客戶端
8.6 SVN 棄用 棄用 舊版版本控制系統，已由 GitHub 取代

9. Data Management（資料管理）

透過 Azure 雲端服務管理項目資料、資料庫與檔案儲存；Unity 客戶端透過 API 讀寫遠端資料。

序號 組件 類型 狀態 說明
9 Data Management 核心分支 使用中 資料管理總類；匯入 Unity 的雲端資料服務
9.1 Azure Static Web 組成部分 使用中 透過 API 建立可視化資料管理介面，支援 CRUD 操作
9.2 Azure Database 組成部分 使用中 Azure Database Flexible Server 遠端 MySQL；開發／測試／正式環境分別對應 VTM-FRP、VTM-CPP
9.3 AWS Web Services 棄用 棄用 舊版 Web 管理介面，已由 Azure Static Web 取代
9.4 AWS Database 棄用 棄用 舊版遠端 MySQL，已由 Azure Database 取代
9.5 Azure Blob Storage 組成部分 使用中 彈性、安全且具成本效益的雲端儲存（vtm_storage）
9.6 AWS S3 Storage 棄用 棄用 舊版雲端儲存，已由 Azure Blob Storage 取代
